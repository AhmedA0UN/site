1.1 ما هي الخوارزمية؟

💡 التعريف الرسمي

الخوارزمية هي مجموعة محدودة من الخطوات المرتبة والمحددة بوضوح لحل مشكلة معينة. تأخذ مدخلات وتنتج مخرجات خلال عدد منتهٍ من الخطوات.

فكّر في الخوارزمية كوصفة طبخ — لديك مكوّنات (مدخلات)، وخطوات محددة (المعالجة)، وطبق جاهز في النهاية (مخرجات). الفرق أن الخوارزمية تحل مشاكل حسابية.

1.2 خصائص الخوارزمية الجيدة

✦
المحدودية (Finiteness): تنتهي بعد عدد محدود من الخطوات.
✦
الوضوح (Definiteness): كل خطوة محددة بدقة، لا غموض.
✦
الفعالية (Effectiveness): كل خطوة قابلة للتنفيذ فعلياً.
✦
المدخلات (Input): صفر أو أكثر من القيم المدخلة.
✦
المخرجات (Output): قيمة ناتجة واحدة على الأقل.
✦
العمومية (Generality): تحل نوع المشكلة، لا حالة واحدة.

CHAPTER 02 — COMPLEXITY ANALYSIS
تعقيد الوقت والمساحة
Big-O Notation
قبل أن تكتب سطراً واحداً من كودك، لازم تعرف كيف تقيس كفاءته. Big-O هي لغة التواصل بين المبرمجين لوصف أداء الخوارزميات.

2.1 ما هو Big-O؟

💡 التعريف

Big-O Notation هو تدوين رياضي يصف كيف يتزايد وقت تنفيذ الخوارزمية أو استهلاكها للذاكرة مع زيادة حجم المدخلات (n). نركز على السلوك عند n الكبيرة ونتجاهل الثوابت.

2.2 ترتيب التعقيدات (من الأسرع للأبطأ)

O(1) → O(log n) → O(n) → O(n log n) → O(n²) → O(2ⁿ) → O(n!)
التعقيدالاسمn=10n=100n=1,000n=10,000مثال
O(1)ثابت1111arr[i]
O(log n)لوغاريتمي371013Binary Search
O(n)خطي101001K10KLinear Search
O(n log n)شبه خطي3366410K133KMerge Sort
O(n²)تربيعي10010K1M100MBubble Sort
O(2ⁿ)أسي1,024∞∞∞Fibonacci Naive
O(n!)عاملي3.6M∞∞∞Permutations

2.3 قواعد حساب Big-O

1
تجاهل الثوابت
O(3n) = O(n) | O(100) = O(1) | O(5n²) = O(n²)
2
احتفظ بالحد الأعلى فقط (Dominant Term)
O(n² + n) = O(n²) | O(n log n + n) = O(n log n)
3
الحلقات المتداخلة تُضرب
حلقة O(n) بداخل حلقة O(n) = O(n²)
4
الحلقات المتتالية تُجمع
O(n) + O(m) = O(n + m) | لكن إن كانت نفس المتغير: O(n) + O(n) = O(n)

2.4 أمثلة عملية بـ C++

C++ — أمثلة Big-Ocomplexity_examples.cpp
// ── O(1) — الوصول المباشر ──────────────────────
int getFirst(vector<int>& arr) {
    return arr[0];  // دائماً خطوة واحدة بغض النظر عن n
}

// ── O(n) — حلقة بسيطة ─────────────────────────
int findMax(vector<int>& arr) {
    int maxVal = arr[0];
    for (int x : arr)          // n تكرار
        maxVal = max(maxVal, x);
    return maxVal;
}

// ── O(n²) — حلقتان متداخلتان ──────────────────
bool hasDuplicate(vector<int>& arr) {
    int n = arr.size();
    for (int i = 0; i < n; i++)       // n تكرار
        for (int j = i+1; j < n; j++)  // n تكرار
            if (arr[i] == arr[j]) return true;
    return false;
}

// ── O(log n) — النصف في كل مرة ────────────────
int binarySearch(vector<int>& arr, int target) {
    int l = 0, r = arr.size() - 1;
    while (l <= r) {              // log n تكرار فقط
        int mid = l + (r - l) / 2;
        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) l = mid + 1;
        else r = mid - 1;
    }
    return -1;
}

2.5 تعقيد المساحة — Space Complexity

💾 تعقيد المساحة

كمية الذاكرة الإضافية التي تحتاجها الخوارزمية. O(1) = لا ذاكرة إضافية. O(n) = ذاكرة بحجم المدخلات.

✦
O(1): Bubble Sort — يعمل في مكانه
✦
O(n): Merge Sort — مصفوفة مساعدة
✦
O(n): Recursion — Call Stack
✦
O(log n): Binary Search (recursive)
🎯 أسئلة مقابلات شائعة — Big-O
ما تعقيد: for(i=0;i<n;i++) for(j=i;j<n;j++) ؟ — الجواب: O(n²) لأن مجموع التكرارات = n(n+1)/2 ≈ n²/2
هل O(2n) = O(n)؟ — نعم، الثوابت تُتجاهل في Big-O تماماً
ما الفرق بين Best/Average/Worst Case؟ — نأخذ Worst Case للضمان، Average للواقع العملي
📝 تمرين — حدد التعقيدEasy
C++ — ما تعقيد كل دالة؟
// (أ) ما تعقيد هذه الدالة؟
void foo(int n) {
    for (int i = 1; i <= n; i *= 2)
        cout << i;
}

// (ب) ما تعقيد هذه الدالة؟
void bar(int n) {
    for (int i = 0; i < n; i++)
        for (int j = 0; j < 100; j++)
            cout << i + j;
}

// (ج) ما تعقيد هذه الدالة؟
void baz(int n) {
    for (int i = 0; i < n; i++)
        cout << i;
    for (int j = 0; j < n*n; j++)
        cout << j;
}

الإجابات:
(أ) O(log n) — i يتضاعف في كل مرة (1,2,4,8...) → log₂n تكرار
(ب) O(n) — الحلقة الداخلية ثابتة (100 مرة دائماً) → تُتجاهل
(ج) O(n²) — الحلقة الثانية O(n²) تسود على O(n) الأولى

3.1 ما هي المصفوفة؟

💡 التعريف

المصفوفة هي مجموعة من العناصر من نفس النوع مخزّنة في مواقع ذاكرة متجاورة. يُحدد كل عنصر بفهرسه (index) الذي يبدأ من صفر.

3.2 عمليات المصفوفة بـ C++

C++arrays_basics.cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    // ── إنشاء مصفوفة ──────────────────────────────
    int arr[5] = {10, 20, 30, 40, 50};   // ثابتة الحجم
    vector<int> v = {10, 20, 30};        // ديناميكية
    vector<int> v2(5, 0);               // 5 عناصر قيمتها 0

    // ── الوصول O(1) ───────────────────────────────
    cout << arr[2];           // 30
    cout << v.front();        // 10 — أول عنصر
    cout << v.back();         // 30 — آخر عنصر
    cout << v.size();         // 3

    // ── الإضافة ───────────────────────────────────
    v.push_back(40);         // O(1) amortized — للنهاية
    v.insert(v.begin(), 5);  // O(n) — للبداية (إزاحة)
    v.insert(v.begin()+2, 15); // O(n) — للوسط

    // ── الحذف ─────────────────────────────────────
    v.pop_back();             // O(1) — من النهاية
    v.erase(v.begin());      // O(n) — من البداية
    v.clear();               // O(n) — حذف الكل

    // ── الاستعراض ─────────────────────────────────
    for (int i = 0; i < v.size(); i++)
        cout << v[i] << " ";

    for (int x : v) cout << x << " "; // Range-based
}

مسألة 1: Two SumEasy GoogleAmazonMetaApple

المشكلة: أعطيت مصفوفة nums وعدد target، أرجع فهارس العنصرين اللذين مجموعهما يساوي target.

مثال: nums = [2,7,11,15], target = 9 → Output: [0,1] لأن 2+7=9

💭فكّر أولاً: الحل Brute Force بحلقتين O(n²). لكن كيف نوصل لـ O(n) باستخدام Hash Map؟ الفكرة: لكل رقم، ابحث عن "complement = target - رقم" في الـ map.
C++ — الحل الأمثل O(n) / Space O(n)
vector<int> twoSum(vector<int>& nums, int target) {
    unordered_map<int, int> seen; // value → index

    for (int i = 0; i < nums.size(); i++) {
        int complement = target - nums[i];

        if (seen.count(complement))
            return {seen[complement], i}; // وجدناه!

        seen[nums[i]] = i; // خزّن لاحقاً
    }
    return {}; // لا حل
}
مسألة 2: Maximum Subarray (Kadane's Algorithm)Medium MicrosoftAppleAmazon

المشكلة: أوجد المصفوفة الجزئية المتجاورة ذات المجموع الأعلى.

مثال: [-2,1,-3,4,-1,2,1,-5,4] → 6 (المصفوفة [4,-1,2,1])

C++ — Kadane's Algorithm O(n) / Space O(1)
int maxSubArray(vector<int>& nums) {
    int maxSum = nums[0];
    int curSum = nums[0];

    for (int i = 1; i < nums.size(); i++) {
        // هل أبدأ من الصفر أم أكمل التسلسل؟
        curSum = max(nums[i], curSum + nums[i]);
        maxSum = max(maxSum, curSum);
    }
    return maxSum;
}
// الفكرة: إذا curSum أصبح سالباً، ابدأ من جديد
مسألة 3: Product of Array Except SelfMedium MetaAmazonGoogle

المشكلة: أرجع مصفوفة بحيث output[i] = حاصل ضرب كل العناصر عدا nums[i]. بدون استخدام القسمة!

مثال: [1,2,3,4] → [24,12,8,6]

C++ — Prefix & Suffix Products O(n) / Space O(1)
vector<int> productExceptSelf(vector<int>& nums) {
    int n = nums.size();
    vector<int> res(n, 1);

    // مرور أول: ضرب ما على اليمين (prefix)
    int prefix = 1;
    for (int i = 0; i < n; i++) {
        res[i] = prefix;
        prefix *= nums[i];
    }

    // مرور ثانٍ: ضرب ما على اليسار (suffix)
    int suffix = 1;
    for (int i = n-1; i >= 0; i--) {
        res[i] *= suffix;
        suffix *= nums[i];
    }
    return res;
}
